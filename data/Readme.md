# SleepSense — Data Modalities & Structure

This directory houses the original StudentLife raw sensor dataset and the preprocessed output generated by the SleepSense ETL pipeline.

---

## Directory Structure

```
data/
├── raw/                         StudentLife original files (untouched)
│   ├── EMA/                     EMA sleep and stress self-report surveys
│   ├── app_usage/               Foreground package usage logs
│   ├── sensing/                 Continuous sensing CSV logs
│   │   ├── activity/            Physical activity states (stationary, walking, running)
│   │   ├── phonelock/           Screen lock/unlock logs
│   │   ├── audio/               Ambient audio silence/conversation labels
│   │   ├── gps/                 Latitude and longitude logs
│   │   └── ...
│   └── notes.csv                [NEW] Simulated student daily text diaries for NLP
│
└── preprocessed/                Pipeline output files (generated)
    ├── features/                42 daily feature vector CSVs per user
    ├── targets/                 Aligned daily sleep quality ratings per user
    └── merged/                  Single unified model-ready dataset (dataset.parquet)
```

---

## Modalities & Feature Engineering

### 1. Daily Sleep Quality Target (EMA/response/Sleep)
- **Target**: Continuous score in range `0.0` (Very bad) to `3.0` (Very good) extracted from the daily sleep EMA `rate` field.

### 2. Physical Activity (sensing/activity)
- Extracted features: `stationary_ratio`, `walking_minutes`, `running_minutes`, `exercise_detected`, `peak_activity_hour`, and `activity_bout_count`.

### 3. Screen Time & Interactivity (sensing/phonelock)
- Extracted features: `unlock_count_late_night` (after 22:00), `unlock_count_evening`, `unlock_count_daytime`, `first_unlock_hour`, `last_unlock_hour`, `avg_session_duration_min`, and `screen_sessions_count`.

### 4. Foreground App Usage (app_usage)
- Categorizes packages into Study, Entertainment, Social, and System apps.
- Extracted features: `app_social_min`, `app_entertainment_evening_min`, `app_late_night_min`, `last_active_app_hour`, `app_diversity_count`, and `app_study_sessions`.

### 5. Social & Ambient Sound (sensing/audio)
- Extracted features: `silence_ratio`, `conversation_ratio`, and `social_audio_evening` (after 19:00).

### 6. Mobility Patterns (sensing/gps)
- Extracted features: `location_entropy` (spatial distribution), `mobility_radius` (travel spread), and `unique_locations_count`.

### 7. Natural Language Processing (raw/notes.csv)
- Dynamic text diary sentences synthesized from physical activity and screen use patterns to simulate student diaries.
- Processed using tokenization, stop-word removal, and **Word2Vec embeddings** to extract `nlp_caffeine_similarity`, `nlp_screen_similarity`, and `nlp_stress_similarity`.
